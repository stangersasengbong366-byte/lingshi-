# 权益清单日常维护说明

## 页面分工

- 销售专用页：链接带 `?sales=1`，只保留产品、科目选择与清单预览，不显示后台入口。
- 用户分享页：销售选择产品和科目后复制分享链接，用户先看到信封，开启后查看权益清单。
- 后台配置页：运营上传课程表、维护赠课与实物资料，先保存草稿，确认后再发布。

## 每季度更新流程

1. 在后台选择对应年级和产品。
2. 01 正课权益分别上传“学法直播”和“知识视频”表格。一个文件可包含 9 个科目，系统按工作表名称自动拆分。
3. 02 赠课权益从年级赠课池勾选。重复赠课只上传一次，其他同年级产品直接复用并设置赠送规则。
4. 03 教辅资料按销售选择的科目自动匹配；买几科就展示几科资料。
5. 点击“保存草稿”仅更新运营后台。
6. 检查销售预览无误后，点击“发布到销售端”。已经发出的销售链接会自动读取最新发布版本。

## 清单阅读逻辑

- 首屏先显示一页权益总览：所购科目、直播数量、知识视频数量、赠课数量和资料数量。
- 每个科目的学法直播与知识视频独立折叠，点击后展示完整大纲。
- 多科不会横向压缩；每个科目均使用完整宽度表格。
- 没有知识视频的科目不显示知识视频表格。
- 赠课和实物资料默认收起，用户需要时再展开查看。

## Supabase 首次配置

首次使用云端同步时，在 Supabase 的 SQL Editor 中完整执行：

`supabase/schema.sql`

执行成功后刷新后台。状态显示“云端草稿已同步”即代表连接完成。

## 飞书接入建议

飞书多维表格适合作为季度数据录入入口，建议建立以下数据表：

1. 产品表：产品编号、年级、卡型、阶段、在售状态、服务期。
2. 正课表：产品编号、科目、课程类型、课次、日期、时间、课程名称、难度。
3. 赠课库：赠课编号、名称、价值、课程明细、适用年级。
4. 产品赠课规则表：产品编号、赠课编号、是否赠送、买一科/两科及以上/买赠对应学科。
5. 教辅资料表：年级、阶段、科目、资料名称、封面图片、赠送规则。

飞书只负责录入与协作，Supabase 负责给销售页和用户分享页提供稳定数据。飞书自动同步需要另行提供飞书应用的 App ID、App Secret、多维表格 App Token 和各数据表 ID。
